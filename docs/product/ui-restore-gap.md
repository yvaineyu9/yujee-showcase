## 🔴 跨屏系统性差距(最高优先,影响所有登录后屏幕)

当前代码库存在**两套割裂的视觉语言**:

| | 视觉语言 A「营销/token 体系」 | 视觉语言 B「朴素 serif 应用面」 |
|---|---|---|
| 用在 | `/`、`/pricing`、`/privacy`、`/terms`(走 `MarketingChrome`) | `/create`、`/my-albums`、`/album/[uuid]`、`/credits`、`/settings`(裸 `<main>`) |
| 标题字体 | sans 无衬线 display | `serif` 衬线 |
| 主按钮 | 黑底 `var(--fg)` pill | 紫底 `var(--accent)` pill |
| 导航 chrome | 有顶栏 + footer | **完全没有任何导航**——裸 main,用户进了 /create 或 /settings 后无处可去 |
| 卡片/容器 | `shadow-card` 柔和阴影圆角 | 1px 边框方框 |

**设计稿的事实**:`yujee-home2.html` 和 `editor.html` 在**每一屏**都有完整登录态 chrome:
1. **顶栏**:brand + 语言选择器 + **credits pill(积分余额 + 套餐名)** + **用户头像下拉菜单**(头像首字母、姓名邮箱、当前套餐+升级、管理订阅、反馈与问题、退出登录)。编辑器顶栏另有可编辑相册标题 + "已保存"脉冲徽章 + 导出 PDF 主按钮。
2. **左侧悬浮 pill 侧边栏**:毛玻璃圆角竖条,主页 / 我的项目 两个图标按钮,是 home↔projects 主切换器,pricing 稿里也有。

**当前实现**:登录后应用面(create/albums/editor/credits/settings)**既没有 credits pill + 用户菜单,也没有左侧侧边栏**。用户看不到积分/套餐,也无统一导航。这是差距最大处,且直接影响转化(看不到积分→不知何时充值/升级)。

> 注:`/login`、`/register` 是登录前页面,用各自的极简 auth 顶栏(brand + 语言),**不套**这套登录态 chrome——这是对的,不算差距。

- **优先级:P0**(同时拖累 create / editor / albums 三个核心转化屏)
- **是否需补设计稿:否**(home/editor 稿已完整定义)
- 给架构:应作为**共享布局组件**统一切活,不要每屏各做一遍(见末尾分工:这是基础流 A,其余流依赖它)。

## [相册编辑器] (路由 `/album/[uuid]`)

- **设计稿**:`editor.html`
- **当前实现**:`app/album/[uuid]/page.tsx` + `components/album/AlbumEditor.tsx`(+ ChatPanel / PageEditPanel / PagePreview)

差距:
1. **核心预览形态**:设计稿中心是**仿真翻书**(1.414:1 跨页对开、中缝阴影、3D perspective + 翻页动画),下方悬浮**毛玻璃缩略图条**(prev/next + 页码 + 跨页缩略图)。当前是**扁平的页面缩略图网格**(点选加紫 ring),**没有书的隐喻、没有跨页、没有翻页动画、没有悬浮缩略图条**。编辑器最大差距。
2. **顶栏缺失**:设计稿有完整顶栏(返回 + brand + 面包屑 + **可编辑相册标题** + **"已保存"脉冲徽章** + 导出 PDF + credits pill + 用户菜单);当前只有简单 header(衬线标题 + prompt 斜体 + 导出),无面包屑/保存态/积分/用户菜单,标题不可编辑。
3. **AI 对话面板** ⚠️推断:设计稿右栏是富对话(AI 头像"己" + 章节上下文 pill + **思考指示器**三色渐变脉冲点 + AI 文字无气泡/用户有气泡 + 引用卡 `ai-quote` + 操作 chips「采纳/再写一版/我来改」+ 章节命名 `chip-select`)。当前 `ChatPanel` 是白底方框——**我没读完该子组件**,"没有思考指示器/引用卡/chips"是推断,待核对。
4. **聊天输入框** ⚠️推断:设计稿是 pill 圆角条 + 圆形发送 + focus 紫光晕;当前疑为普通框,待核对。
5. **布局**:设计稿 `1fr 340px` 左书右聊,无独立"页面编辑表单";当前 `1fr 360px`,左缩略图网格 + 右栏(编辑面板 + 聊天上下叠)。
6. **视觉语言**:设计稿满屏 token + 无衬线 display;当前 serif 标题 + 米色 paper 块,偏"纸感 demo"。

- **优先级:P0** ·**是否需补稿:否**(editor 稿很完整,含动效)
- 给架构:翻书 3D 动画 + 思考指示器是这屏的"灵魂",工作量大但不可省。

# 关于"拆 worktree / 分工作流"

本文档是本版**「修页面 / UI 还原」这一整块需求**的完整拆解(逐屏差距 + 优先级已在上)。除了"修页面",本版可能还有别的工作(非 UI),那是另一块。

**要不要把这些需求拆成多个 worktree、怎么拆、谁修哪屏——交给项目经理 + 架构师决策,不是 PM 的活。** 这里只补一个对他们有用的事实:**统一登录态 chrome(顺序表第 1 项)是基础**,编辑器/首页/作品列表/积分页/设置页都消费它;若并行开工,这块最好先落地或先定好组件接口,免得多人同时改同一个布局组件起冲突。
